解决在Windows下搭建Gitlab服务器不能访问Gitlab的web管理页面的问题

  • 2020
  • 03
  • GITLABWEB

两个问题：

1.Gitlab搭建好之后，不能通过http://localhost或者http://127.0.0.1访问Gitlab的web管理端页面。

2.自定义Gitlab域名，如gitlab.abc.com

针对第一个问题，查看80端口是否被占用，我遇到的问题，就是80端口被apache给暂用了。针对此问题，直接停用apache服务，或者修改apache的默认端口。

然后重启gitlab就可以访问。

第二个问题，需要修改两个文件，分别是：/etc/gitlab/gitlab.rb文件和/var/opt/gitlab/gitlab-rails/etc/gitlab.yml

首先，设置hosts文件映射：127.0.0.1 gitlab.abc.com

然后，修改 /etc/gitlab/gitlab.rb，此文件需要修改成external ‘http://gitlab.abc.com'

————————————————

/var/opt/gitlab/gitlab-rails/etc/gitlab.yml，此文件，找到## Web server settings

将host的值改成：gitlab.abc.com

最后gitlab的web管理页面就可以正常访问，并通过自定义域名访问了。
